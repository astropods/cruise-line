# Fix fork PR cloning and resolve TypeScript errors

## Summary

Pull requests from forks failed with "Remote branch not found" because the clone
logic used `git clone --branch <headRef>` against the **base** repository. For fork
PRs the head branch only exists on the fork, not the upstream repo. This also fixes
all pre-existing TypeScript strict-mode errors across the agent codebase.

## Design

### Fork-safe cloning via PR refs

GitHub always creates `refs/pull/<number>/head` on the base repository for every PR,
regardless of whether it originates from a fork or the same repo. The sandbox
`/ensure-clone` endpoint now:

1. Clones the base repo without `--branch` (default branch only).
2. Fetches `+refs/pull/<N>/head:refs/remotes/origin/pr-head` to pull the PR's head
   commit into the local clone.
3. Checks out by exact SHA (`git checkout <headSha>`).

This replaces the old `--single-branch --branch <headRef>` strategy which assumed the
head branch lived on the base repo's origin. The same pattern is applied to the
update path (existing clone at wrong SHA) and the self-heal retry path.

A new `prNumber` field was added to `EnsureCloneParams` and threaded through from
`analyzer.ts` and `chat.ts` to the sandbox endpoint.

### Strict TypeScript without type overrides

All `as` casts, `!` non-null assertions, and `as never` workarounds were removed in
favour of proper type-level fixes:

- **`AppEnv` type** (`agent/env.ts`) — defines `Variables: { session: SessionPayload }`
  and is used by all Hono route files and middleware so `c.get('session')` returns the
  correct type without casting.
- **Generic rate limiter** — `rateLimit<E extends Env>()` propagates the app's env
  type into the `keyFn` callback, giving it a properly typed context.
- **`interface` to `type` for analysis types** — `Walkthrough`, `Finding`, and
  `FileContent` were converted from interfaces to type aliases. Type aliases carry
  implicit index signatures in TypeScript, which satisfies postgres.js's `JSONValue`
  constraint without casting.
- **Runtime guards for route params** — where Hono's inline middleware breaks path-based
  param inference (returning `string | undefined`), truthiness guards narrow the type
  naturally instead of using `!`.

## Migration

No migration required. The `prNumber` field is optional on the sandbox endpoint for
backward compatibility, though all current callers now provide it.
