# Add Claude Opus 4.8 as a model option

## Summary

Anthropic released Claude Opus 4.8 (`claude-opus-4-8`), which is now the recommended top-tier model for "complex reasoning, long-horizon agentic coding, and high-autonomy work" per the [official model docs](https://platform.claude.com/docs/en/docs/about-claude/models). PR walkthrough generation is exactly that workload, so 4.8 is added to the deploy-time `CLAUDE_MODEL` select and promoted to the default. Older Opus snapshots (4.7, 4.6) are kept as selectable options since the docs still list them as available — only the default moves.

## Design

The user-facing model picker lives in `astropods.yml` under the `agent.inputs` section as a `display-as: select` input. The Astropods deploy flow renders that schema, so updating the `options:` list and `default:` is sufficient to expose the new model choice in the deploy UI.

```yaml
- name: CLAUDE_MODEL
  options:
    - claude-opus-4-8   # new — current top model
    - claude-opus-4-7
    - claude-opus-4-6
    - claude-sonnet-4-6
    - claude-haiku-4-5
  default: "claude-opus-4-8"
```

The agent runtime reads the same env var in `agent/config.ts`. Its fallback default is synced to `claude-opus-4-8` so local dev (which doesn't go through the Astropods deploy form) lands on the current model out of the box. Doc tables in `README.md` and `AGENT.md` are updated to match — the README had also drifted to a stale `claude-opus-4-5` reference, which is corrected here.

### Source of truth

The current model list was verified against `https://platform.claude.com/docs/en/docs/about-claude/models` rather than recalled from training context. The docs treat 4.8 as the current Opus and move 4.7 / 4.6 into the legacy table; both remain available via the API, so they stay in the dropdown for users with reasons to pin an older snapshot (cost, behavior parity with an in-flight investigation, etc.).

## Migration

Nothing required for existing deploys — `CLAUDE_MODEL` is already populated in their saved config and will keep using whatever model they previously selected. New deploys, and any deploy that re-saves without overriding, will pick up Opus 4.8 as the default.
